# Политика безопасности

## Поддерживаемые версии

Безопасностные исправления выпускаются для последней мажорной ветки.

| Версия    | Поддерживается |
|-----------|----------------|
| 2.1.x     | Да             |
| 2.0.x     | Нет, обновитесь до 2.1.x |
| < 2.0     | Нет            |

## Как сообщить об уязвимости

**Не публикуйте уязвимости в issues и не раскрывайте их публично.** Сначала
свяжитесь со мной приватно:

- **Email:** `nizamidrisov98@gmail.com`
- В теме письма: `[SECURITY] Yandex Mail MCP`

Опишите:

1. Что обнаружено (тип уязвимости — утечка данных, обход подтверждения,
   prompt-injection, проблема с криптографией и т.д.).
2. Как воспроизвести (минимальный пример).
3. Возможные последствия (что атакующий мог бы получить).
4. Если есть — предлагаемый патч.

Я отвечу в течение 72 часов. После подтверждения уязвимости и выпуска
исправления — обсудим публичное раскрытие (обычно через 30-90 дней после
выпуска патча, в зависимости от серьёзности).

## Что не считается уязвимостью

- **Получение Яндексом метаданных писем.** Сервер использует официальный
  Яндекс-IMAP/SMTP — данные ходят через инфраструктуру Яндекса по своим
  правилам, наш сервер этого не меняет.
- **AI-ассистент видит содержимое прочитанных писем.** Это by design —
  чтение писем и есть его задача. Защита нацелена на то, чтобы AI не мог
  **отправить** что-то опасное без вашего подтверждения.
- **Кнопка "Always allow" в клиенте отключает защиту.** Наша защита
  работает на стороне сервера и не зависит от поведения клиента — даже с
  включённым "Always allow" одноразовый код всё равно нужен для отправки.
