# Как помочь проекту

Спасибо, что интересуетесь. Несколько правил.

## Сообщить о баге

Открой issue, опиши:
- Что ожидалось.
- Что произошло на самом деле.
- Версия (`yandex-mail-mcp --version` или содержимое тега).
- Версия клиента MCP (Claude Desktop / Claude Code / Cursor / другой).
- ОС и версия Node.js (`node --version`).
- Если есть — фрагмент лога (без чувствительных данных, см. ниже).

**Не публикуйте** содержимое:
- `token.json` (логин, пароль приложения).
- `audit.jsonl` (метаданные могут косвенно раскрыть переписку).
- `allowlist.json` (адреса контактов).

## Предложить улучшение

Открой issue с описанием задачи, желательно с конкретным сценарием
использования. Перед началом разработки обсудим — это сэкономит время.

## Прислать pull request

1. Форкни репозиторий.
2. Создай ветку от `main`: `git checkout -b fix-описание-кратко` или
   `feature/описание-кратко`.
3. Сделай изменения. Перед коммитом запусти проверки:
   ```bash
   cd yandex-mail-mcp-desktop
   npm test
   npm audit --omit=dev
   npm run build
   ```
   Все три должны пройти без сбоев.
4. Коммит с понятным сообщением. Формат: `<тип>: <что>`, например:
   `fix: исправлен race при первом запуске`. Возможные типы: `feat`, `fix`,
   `docs`, `test`, `refactor`, `chore`.
5. Открой pull request с описанием: что меняется, зачем, как тестировал.

## Сообщить уязвимость

Не через публичный issue. См. [SECURITY.md](SECURITY.md).

## Стиль кода

- TypeScript строгий (`strict: true`), без `any`. Используй `unknown` или
  правильные типы.
- Импорты с расширением `.js` (это правила ESM/NodeNext).
- Никаких эмодзи в исходниках.
- Комментарии только там, где **почему**, а не **что** — имена переменных и
  функций должны говорить сами за себя.

## Что НЕ принимаю

- Новые зависимости без серьёзного обоснования (минимум зависимостей —
  принцип проекта).
- Изменения, которые ослабляют защиту от prompt-injection или меняют
  серверную модель доверия на клиентскую.
- "Косметические" PR'ы только ради активности (правка пробелов,
  переименования без причины, etc.).
