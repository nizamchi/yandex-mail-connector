# `/ymc-index` — поисковый индекс Yandex Mail коннектора

Мгновенный поиск (`yandex_search_fast`) и каталог вложений (`yandex_find_attachments`)
работают по локальному индексу. Пока индекс не собран — они возвращают пусто.
Эта команда собирает и обновляет индекс из чата, без похода в терминал.

Пиши по-русски, коротко, по делу. Сначала покажи состояние, потом дай выбрать действие.

## Шаг 1 — найти бандл

Прочти `~/.claude.json` (Windows: `%USERPROFILE%\.claude.json`). Найди
`mcpServers.yandex-mail.args` — массив; первый элемент `args[0]` это абсолютный путь
к `dist/yandex-mail-mcp.js`. Это и есть бандл, который запускает индексатор:
`node "<args[0]>" index <команда>`.

- Если запись не найдена — скажи: «Сервер `yandex-mail` не настроен в `~/.claude.json`».
- Если файла по пути нет — предложи `/ymc-update` или переустановку.

(Если у пользователя несколько серверов — `yandex-mail-work` и т.п. — спроси, для какого
аккаунта собрать индекс, и возьми его `args[0]`. У каждого аккаунта индекс свой.)

## Шаг 2 — показать текущее состояние

Запусти `node "<args[0]>" index status`. Покажи кратко: индекс есть/пуст, аккаунт,
сколько писем и в каких папках, готовы ли цепочки (threads). Счётчик вложений в
выводе — это всего по этому state-dir (по всем аккаунтам сразу), а не по одному
аккаунту; так и подай, не выдавай за счёт одного ящика.

## Шаг 3 — выбрать действие

Через `AskUserQuestion` (header «Индекс»):

1. **Собрать / пересобрать** — полный обход папок. Спроси какие: предложи
   «INBOX + Отправленные» (рекомендую — нужно для цепочек писем), «только INBOX»,
   или свой список. Запусти `node "<args[0]>" index build INBOX Sent` (или с выбранными
   папками).
2. **Обновить (досинхронизировать)** — добрать новые письма в уже собранные папки.
   Запусти `node "<args[0]>" index update` (без аргументов — обновит все
   проиндексированные папки). Если индекс собирали до версии с вложениями
   (v2.9.0), `update` заодно до-наполнит каталог вложений в старых папках.
3. **Показать статус** — просто повтори вывод Шага 2.
4. **Удалить индекс** — спроси подтверждение, затем `node "<args[0]>" index drop`.

## Шаг 4 — отчёт

Покажи результат прогона (сколько писем добавлено, ошибки если были) и свежий статус.

Скажи честно: **перезапуск Claude Code НЕ нужен.** Индекс — это файл; работающий
сервер перечитывает его при следующем поиске. Это не как `/ymc-config` (там уровень
доступа читается один раз при старте).

Если папок много, сборка идёт по сети и занимает время — предупреди заранее. Большие
ящики собирай по папкам, а не всё разом.

## Чего НЕ делать

- Не вызывай MCP-инструменты почты ради этого — индексатор это отдельный процесс
  (`node "<args[0]>" index ...`), он сам ходит в IMAP.
- Не собирай молча все папки подряд на большом ящике — сначала уточни у пользователя.
- Не показывай содержимое писем — индекс это служебные данные, а не чтение почты.
