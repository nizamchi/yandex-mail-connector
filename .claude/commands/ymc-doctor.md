# `/ymc-doctor` — диагностика Yandex Mail коннектора

Пользователь пришёл с «не работает» / «не подключается» / «не отправляет». Твоя
задача — прогнать проверки и объяснить простым языком, что зелёное, что красное и
как починить каждое красное.

Это НЕ обзор возможностей (для этого `/ymc-info`). Это проверка работоспособности:
сеть, учётные данные, подписи, версия.

## Шаг 1 — найти бандл

Прочти `~/.claude.json` → `mcpServers.yandex-mail.args[0]` — абсолютный путь к
`dist/yandex-mail-mcp.js`. Если не нашёл — скажи, что сервер не настроен, и покажи
строку установки (`claude mcp add yandex-mail ...`).

## Шаг 2 — сетевой health-check (главное)

Запусти `node "<args[0]>" --check`. Он проверяет **без входа в почту** (не оставляет
следов «был выполнен вход в аккаунт»):

- где лежит состояние (state-dir);
- тип учётных данных в `token.json` (пароль приложения или OAuth) — без показа значения;
- валидна ли подпись allowlist;
- читается ли политика рисков;
- доступны ли `imap.yandex.com:993` и `smtp.yandex.com:465` по TLS.

Код выхода 0 — всё ок; 1 — хоть один пункт упал. Покажи вывод и разбери его.

## Шаг 3 — состояние работающего сервера

Если инструмент `yandex_health_check` доступен — вызови его (уровень 0, без сети).
Он покажет, на какой версии и уровне доступа работает **запущенный** сервер прямо
сейчас. Сравни версию с той, что в бандле: если запущенный сервер старее — нужен
перезапуск Claude Code, чтобы подхватить новый бандл (или `/ymc-update`).

## Шаг 4 — разбор и починка

Сведи всё в короткий отчёт «✓ зелёное / ✗ красное» и для каждого красного дай
конкретный шаг:

- **IMAP/SMTP недоступны** → проверь интернет/файрвол/VPN; порты 993 и 465 могут быть
  закрыты в корпоративной сети.
- **token не найден** → не заданы учётные данные. Нужен пароль приложения Яндекса
  (passport.yandex.ru → пароли приложений), положить в `token.json` или в env
  (`YANDEX_APP_PASSWORD` + `YANDEX_EMAIL`).
- **подпись allowlist невалидна** → файл или ключ меняли вручную; покажи варианты
  восстановления (удалить `allowlist.json` для пересборки из «Отправленных», либо
  удалить и его, и `secret.bin` — это сотрёт все доверенные адреса).
- **версия запущенного сервера старее бандла** → перезапусти Claude Code (`/exit` →
  `claude`); если и бандл устарел — `/ymc-update`.
- **нет нужного инструмента (отправка/удаление)** → это не поломка: уровень доступа
  ниже требуемого. Поднять — `/ymc-config` + перезапуск.

Если всё зелёное — так и скажи: «коннектор исправен, связь есть, учётные данные на
месте». Не выдумывай проблемы там, где их нет.

## Чего НЕ делать

- Не показывай значения паролей/токенов — `--check` и `health_check` отдают только тип.
- Не делай IMAP-вход ради проверки (не зови `yandex_list_emails` и т.п.) — для этого
  есть `--check`, который намеренно не логинится.
- Не «чини» молча (не трогай `allowlist.json` / `token.json` сам) — покажи варианты,
  решение за пользователем.
